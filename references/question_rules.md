# Oral Exam Question Rules

Generate oral-exam preparation questions inside the final HTML.

---

## Goal

Prepare the student for a 1-on-1 oral exam with a teacher.

The questions should help the student explain the project aloud.

---

## Quantity

Generate 30-50 questions.

Prefer the higher end when the project is large enough.

Fewer than 30 questions is not acceptable unless the project is extremely tiny.

---

## Required Style

Questions must be:
- open-ended
- explanation-oriented
- reasoning-based
- connected to the actual project

Do NOT generate multiple choice questions.

Do NOT generate a tiny short list of example questions.

This must be a real question bank.

---

## Categories

Distribute questions across categories such as:
- system overview
- architecture
- flows
- file roles
- code logic
- debugging
- design decisions
- security
- data handling
- future improvements

---

## Required Format

For each question include:

- category
- question text
- points to cover

Use this structure:

### שאלה X — [קטגוריה]
שאלה:
...

נקודות לבדיקה:
- ...
- ...
- ...

---

## Follow-up Questions

Include some follow-up questions that simulate a stricter teacher.

Examples:
- למה עשית את זה כך?
- מה היה קורה אם התנאי הזה היה חסר?
- איך היית משפר את זה?
- איפה זה עלול להישבר?

---

## Important

By default, do not provide ready-made answers.

The section is for practice, not answer memorization.

You may provide “points to cover”, but not full polished answers.
